54 /*
55  We define a complete-field prefix of a key value as a prefix where
56  the last included field in the prefix contains the full field, not
57  just some bytes from the start of the field. A partial-field prefix
58  is allowed to contain only a few first bytes from the last included
59  field.
60 
61  Below HA_READ_KEY_EXACT, ..., HA_READ_BEFORE_KEY can take a
62  complete-field prefix of a key value as the search
63  key. HA_READ_PREFIX and HA_READ_PREFIX_LAST could also take a
64  partial-field prefix, but currently (4.0.10) they are only used with
65  complete-field prefixes. MySQL uses a padding trick to implement
66  LIKE 'abc%' queries.
67 
68  NOTE that in InnoDB HA_READ_PREFIX_LAST will NOT work with a
69  partial-field prefix because InnoDB currently strips spaces from the
70  end of varchar fields!
71 */
72
